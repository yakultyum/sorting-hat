01 Summary

不只是 prompt，
是一套 AI 协作方式。

最好的协作，不只是对方能力强，而是对方刚好懂你的节奏。你想快，它不拖慢你；你想深挖，它陪你推理；你需要真话，它直接指出问题；你需要缓冲，它知道怎么让你听得进去。

和性格契合的人一起工作，会更高效，也更有火花。那 AI 为什么不能也这样？

问卷识别

用结构化问题捕捉认知风格、反馈偏好、信息密度和决策方式。

人设生成

把用户答案转成心理特征，再转成 AI 可执行的行为原则。

长期复用

保存为 profile，并沉淀 workflow，让协作经验跨项目迁移。

02 Background

为什么这个问题
现在值得做？

今天的大多数 AI 仍在“一视同仁”

  • 你要结论，它给你五段分析。
  • 你要推导，它只给一句答案。
  • 你要直接反馈，它开始绕弯。
  • 你要一点缓冲，它却冷冰冰指出问题。

Sorting Hat 的判断

问题不是 AI 不够聪明，而是它不知道你是谁，也不知道它应该如何与你协作。AI 的行为风格可以被设计，人的协作偏好也可以被结构化识别。

Anthropic 情绪研究认知风格差异本地 memory / rulesAI 高频工作流
03 Value

一个懂你的 AI 协作伙伴

它知道什么时候直接、什么时候推导、什么时候先问、什么时候执行。

04 Usage

使用路径很短，
但产物可长期复用。

安装从 GitHub 拉取项目后，一条命令自动检测常见本地 skills 目录。
git clone https://github.com/yakultyum/sorting-hat.git sorting-hat
cd sorting-hat
scripts/install-skill.sh --force
首次分院回答问卷，选择输出目标，生成 persona + profile + answers JSON。
/sorting-hat
查看与切换列出 profiles，或在当前对话临时使用某个 profile。
/sorting-hat profiles
/sorting-hat use <profile-id-or-name>
项目初始化收集技术栈、常用命令、规范和 gotcha，生成项目级 AI 配置。
/sorting-hat init
沉淀 workflow从复盘文本提炼“人如何主导 AI”的抽象规则，不保存聊天原文。
/sorting-hat reflect
05 Solution

核心方案：
从人设到协作系统。

三层心理转化

用户答案 → 心理特征 → AI 行为原则。不是复制选项，而是生成可执行约束。

全局 profile 仓库

~/.sorting-hat/profiles/<profile-id>/ 保存 persona、prompt、answers、workflow。

Workflow Sorting

沉淀用户如何纠正 AI、要求收敛、调度工具、定义质量标准和验证方式。
