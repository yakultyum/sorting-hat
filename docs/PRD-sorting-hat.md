# PRD · Sorting Hat — AI 人设分院仪式

**版本：** v1.0
**日期：** 2026-06-10
**状态：** 草稿

---

## 1. 摘要

Sorting Hat 是一个运行在 Claude Code 里的 skill，通过一套融合七个心理学框架的结构化问卷，帮助用户找到最适合自己认知风格的 AI 人设，并将结果自动写入本地记忆文件，让 AI 在每次对话中以该人设工作，无需重复设置。它解决的核心问题是：大多数人以完全相同的方式使用 AI，但每个人和 AI 协作的最佳方式其实差异很大，而这种差异不是偏好问题，是认知结构问题。

## 3. 背景

### 3.1 项目故事与灵感来源

这个项目的灵感来自两件同时发生的事，它们碰在一起，产生了一个过去不可能出现的想法。

**第一件事：一篇改变了我们对 AI 认知的论文。**

2026 年，Anthropic 发表了研究论文《Emotion Concepts and their Function in a Large Language Model》。这篇论文的核心发现令人意外：Claude 内部存在 171 种可测量的情绪表征，这些表征不是表面的措辞风格，而是真实影响 AI 输出行为的内部状态——从回答的详细程度，到面对压力时是坚持还是退让，再到给出反馈时是温柔还是直接，都受到这些内部状态的影响。

更重要的是，论文还发现了一个具体的张力：AI 的"温暖感"和"直率程度"之间存在内在冲突——越温暖越容易讨好用户，越直率越可能让用户不舒服。这不是可以靠一句系统提示解决的问题，而是需要精确调校的参数。

这意味着：**AI 的"人格"不是比喻，是一组可以被测量、被设计、被精确配置的参数。**

**第二件事：一个反复出现的困惑。**

在团队里，我们观察到一件奇怪的事：同样的 AI 工具，不同的人用出来的体验差异极大。有人说 Claude 太啰嗦、废话太多；有人说太简短、缺少推导。有人觉得它的反馈太直接、有点冒犯；有人觉得正好、节省时间。有人希望 AI 帮自己发散思路；有人希望它直接给最优解。

起初我们以为这是"AI 水平"的问题，后来发现根本原因不同——这些差异来自每个人不同的**认知风格**。心理学研究早就告诉我们，人在处理信息、做决策、接受反馈时，有深度的个体差异：有人的思维是发散的（Ne 主导），有人是收敛的（Ni 主导）；有人决策时先看风险（预防焦点），有人先看机会（促进焦点）；有人需要情感缓冲才能接受批评（Fe 主导），有人认为直接说才是尊重（Fi 主导）。

这些认知差异不是偏好，是结构性的。**用错误的方式协作，不是 AI 不好用，是人设没配对。**

**两件事叠在一起，产生了 Sorting Hat 的构想。**

既然 AI 的"人格"是可以精确调校的参数集合，而每个人的认知风格是可以被测量的结构性特征，那我们能不能做一个工具，把这两件事连起来——像哈利波特里的分院帽一样，通过一次测试，把你分到最适合你的 AI"学院"，然后这顶帽子记住你，往后的每一天都以最适合你的方式和你工作？

**名字的来源** 就在这里。哈利波特里的分院帽（Sorting Hat）是一顶能读心术的帽子，它了解每个学生的内心特质，把他们分配到最适合发挥潜能的学院。这个隐喻在这里非常贴切——我们的工具不是在给用户贴标签，而是在理解每个人内心真实的认知结构，找到最匹配的 AI 协作方式。而且，就像分院帽一样，分配结果不是永久锁死的，它会随着你的成长而更新。

### 3.2 为什么是现在

这件事之所以现在才能做，是因为 Claude Code 的 memory 系统在技术上解决了"人设持久化"的问题。用户只需要经历一次分院，结果写入 `~/.claude/memory/persona.md`，此后每次打开 Claude Code，这份人设会被自动加载进上下文，无需用户重复设置。这是过去没有的基础设施——有了它，"一次配置，永久生效"才成为可能。

三件事的叠加形成了当下的机会窗口：

1. **Anthropic 情绪论文** 提供了理论依据：AI 人格是可精确调校的参数，不是玄学
2. **Claude Code memory 系统** 提供了技术基础：人设可以持久化，不需要每次重设
3. **团队对 AI 协作的深度需求** 提供了真实场景：员工已经在高频使用 AI，但体验参差不齐

### 3.3 现有方案的问题

目前用户解决"AI 不好用"的方式有两种，都不理想：

- **反复写 system prompt**：每次新对话重新粘贴一段长提示，繁琐且效果不稳定
- **摸索和适应**：花很长时间试错，逐渐发现哪种提问方式效果好，但没有系统化，下次换个任务又要重来

两种方式都缺少一个关键步骤：**从心理学出发，先搞清楚自己的认知风格是什么，再系统性地配置 AI 的行为**。

## 4. 目标

### 4.1 产品目标

帮助高频使用 Claude Code 的员工在 5-20 分钟内完成一次"分院"，生成一份精准反映自己认知风格的 AI 人设档案，并从此之后在每次 AI 协作中以该人设工作，不再因为人设不匹配而感到"AI 不好用"。

### 4.2 为什么重要

- **对用户**：减少 AI 协作的摩擦感，从"调教 AI"变成"直接用 AI"，节省每天在提示词上的认知成本
- **对团队**：让 AI 工具真正在团队中落地，而不是停留在"会用但用得不顺"的阶段
- **对未来**：建立一套可复用的心理学 × AI 配置框架，未来可以扩展到团队级人设、角色级人设

### 4.3 成功指标（关键结果）

| 指标 | 目标值 | 测量方式 |
|---|---|---|
| 分院完成率 | 简易版 ≥ 85%，深度版 ≥ 60% | 开始问卷 vs 写入文件的比例 |
| 人设准确度（主观评分） | ≥ 4/5 分 | 完成后向用户询问"这份人设准确描述了你吗" |
| 首次使用后的人设修改率 | ≤ 30%（30 天内） | `persona.md` 在首次生成后 30 天内被重新覆盖的次数 |
| 种子用户 NPS | ≥ 40 | 首批 10 名用户的净推荐值 |

## 5. 目标用户

### 5.1 主要用户

**高频 AI 协作工作者**——每天或每周多次使用 Claude Code 完成工作任务的员工，包括但不限于：

- 支付业务产品经理（处理需求文档、业务分析、方案评审）
- 技术开发（代码生成、架构设计、问题排查）
- 数据分析师（查询优化、报告撰写、数据解读）

**用户特征：**

- 已经在用 AI，不是 AI 的陌生人
- 对 AI 有明确的工作需求，不是随意闲聊
- 曾经感受过"AI 回答方式不对"的挫败感，但不知道如何系统解决
- 有意愿花 5-20 分钟做一次配置，如果能换来长期的体验提升

### 5.2 非目标用户

- **偶尔使用 AI 的用户**：配置成本和收益不成比例
- **已经有完整 system prompt 体系的高级用户**：他们已经自己解决了这个问题
- **完全不使用 Claude Code 的用户**：本产品依赖 Claude Code 的 memory 系统

### 5.3 用户约束

- 需要安装并使用 Claude Code
- 需要有基础的 AI 使用经验（能理解问卷中的工作情景）
- 中文用户（当前版本，未来可扩展英文版）

## 6. 价值主张

### 6.1 用户要完成的工作（Jobs to Be Done）

| 情景 | 用户的任务 | 当前痛点 |
|---|---|---|
| 每次打开 AI 开始工作 | 快速进入高效协作状态 | 需要重新解释自己的偏好，或容忍不匹配的回答风格 |
| 发现 AI 回答方式"不对" | 调整 AI 的行为 | 不知道该改什么，只能靠感觉反复试错 |
| 把 AI 推荐给同事 | 让同事也能高效使用 AI | 无法简单解释"怎么配置才好用" |

### 6.2 用户将获得什么

- **一次性解决**：花 5-20 分钟做一次问卷，从此不再每次重新解释自己的偏好
- **心理学精度**：不是随机选选项，而是经过七个成熟心理学框架验证的精准匹配
- **可更新**：人格会变，工作场景会变，随时调用 `/sorting-hat` 修改某一个维度，不需要全部重来
- **零负担**：无需安装任何工具，不上传任何数据，结果存在自己的本地机器上

### 6.3 解决了哪些痛点

- **痛点 1：每次对话都要重新调教 AI** → Sorting Hat 一次分院，永久生效
- **痛点 2：不知道为什么 AI 感觉不对** → Sorting Hat 用心理学语言帮你说清楚
- **痛点 3：AI 对所有人说话方式一样** → Sorting Hat 让 AI 记住你是谁

### 6.4 我们比现有方案好在哪里

| 比较维度 | 手写 system prompt | 随机摸索 | Sorting Hat |
|---|---|---|---|
| 配置速度 | 需要自己写，费时费力 | 无，靠时间积累 | 5-20 分钟，一次完成 |
| 配置精度 | 取决于用户自我认知 | 低，靠感觉 | 心理学框架支撑 |
| 持久化 | 每次新对话要重新粘贴 | 无法持久化 | 自动写入 memory，永久生效 |
| 可更新性 | 手动修改，容易遗忘 | 无法系统更新 | 按维度更新，记录历史答案 |
| 背后理论 | 无 | 无 | 七个成熟心理学框架 |

## 7. 解决方案

### 7.1 用户旅程

**首次使用**

```
用户打开 Claude Code
    ↓
调用 /sorting-hat
    ↓
看到欢迎语 + 项目介绍
    ↓
选择简易版（5分钟）或深度版（20分钟）
    ↓
逐题回答（通过点击界面选择，无需手动输入）
    ↓
AI 按三层转化规则生成人设
    ↓
结果写入 ~/.claude/memory/persona.md
    ↓
展示生成的人设摘要
    ↓
下次对话起，AI 自动以该人设工作
```

**再次使用（修改）**

```
调用 /sorting-hat
    ↓
展示当前人设摘要 + 所有维度设置
    ↓
用户选择要修改的维度（或全部重来）
    ↓
只重新回答该维度的问题
    ↓
更新 persona.md 对应部分
    ↓
展示更新后的人设摘要
```

### 7.2 核心功能

#### 功能 1：简易版问卷（11 题，约 5 分钟）

覆盖最核心的人设维度，适合快速上手。包含：
- 推理深度偏好（荣格 Te/Ti + 认知需求）
- 反馈方式偏好（荣格 Fe/Fi）
- 问题解法偏好（荣格 Ne/Ni）
- 情绪底色（Anthropic 情绪研究，效价 × 唤醒度）
- 温暖 vs 直率平衡（Anthropic 情绪研究，讨好倾向）
- 信息密度偏好（认知负荷理论）
- 格式偏好（认知负荷理论）
- 调节焦点（机会 vs 风险呈现顺序）
- 专业领域（开放填写）
- 称呼方式
- 角色融合（可选，引入特定角色的说话风格）

#### 功能 2：深度版问卷（31 题，约 20 分钟）

覆盖全部七个心理学框架，生成高精度人设档案。新增维度：
- 主动介入程度（AI 是否主动指出问题）
- 压力决策模式（紧急情况下如何给答案）
- 认知独立性（AI 被质疑时是否坚持立场）
- 关系定位（同事 / 助理 / 老师 / 顾问）
- 决策框架偏好
- 验证行为（Te/Ti、Fe/Fi、Ne/Ni 的交叉验证题）
- OCEAN 大五人格（开放性、尽责性、神经质）
- 依恋风格（连续性需求、错误容忍度、关系语言）
- 协作节奏（一次交付 vs 多轮对齐）
- 专业背景脚手架需求
- 控制点（执行权给多少）

#### 功能 3：三层心理转化规则

这是 Sorting Hat 区别于普通配置工具的核心设计。每道题的用户答案不直接翻译为人设语言，而是经过三层转化：

```
用户答案（情景选择）
    ↓
心理特征（该选择揭示的认知/情感/行为模式）
    ↓
人设语言（翻译成 AI 的通用行事原则）
```

例：
- 用户选「带我一步步推导」
- 心理特征：Ti 主导 + 高认知需求，被思考过程本身驱动，理解优先于速度
- 人设语言：「暴露完整思维链，不替用户跳过推理步骤；用户真正理解比快速给出答案更重要」

这套设计确保人设是通用的行为原则，而不是绑定在特定情景上的条件反应。

#### 功能 4：人设文件持久化

生成的人设写入 `~/.claude/memory/persona.md`，文件包含两部分：
1. **答题记录**：结构化存储每道题的选择，供下次修改时展示
2. **AI 人设描述**：供 Claude 直接读取的第一人称命令式 system prompt，包含六个层次：
   - **身份与风格**：情绪底色、温暖/直率平衡、角色融合（可选）
   - **专业背景**：领域假设、专业术语默认深度
   - **每次对话的启动姿态**：收到任务时的第一个动作（直接开始 / 先确认 / 先问参与深度）
   - **沟通规则**：推理深度、信息密度、格式偏好、决策分析顺序、称呼方式
   - **反馈与判断**：给负面反馈的方式（温和缓冲 / 直接 / 先理解再评价）
   - **思维方式**：解法风格（发散探索 / 有限选项 / 直指本质 / 问题定义优先）
   - **行为边界（禁止项）**：从答案自动推断的负向约束，比正向描述对 AI 行为的影响更精准（如「不重复用户刚说过的内容」「不用模糊语气掩盖真实判断」）
   - **深度设置**（深度版专属）：主动介入程度、关系定位、执行授权、输出完整性、风险呈现方式

人设采用**第一人称命令式**写法（"我默认给出行动清单，不展开推导"），而非第三人称描述，Claude 的遵从度更高。

#### 功能 5：按维度局部更新

再次调用 `/sorting-hat` 时，用户可以只修改某一个维度（如"我最近更需要直接反馈，把反馈方式改一下"），无需重新完成整个问卷。

### 7.3 技术实现

- **类型**：纯 Markdown skill（`SKILL.md`），零外部依赖
- **交互**：通过 `AskUserQuestion` 工具呈现选项，用户点击选择，无需手动输入
- **视觉**：ANSI 颜色码渲染，终端天然支持，零额外安装
- **文件写入**：通过 Claude Code 的 Write 工具写入本地文件
- **存储**：`~/.claude/memory/persona.md`，数据不离开用户本地机器

**视觉设计规范：**

| 元素 | 颜色 | 用途 |
|---|---|---|
| ASCII 大标题 | 金色 | 开场仪式感 |
| Phase 标题 | 紫色 | 区分阶段 |
| 题目编号 | 金色 | 视觉层次 |
| 选项字母 | 青色 | 引导用户 |
| 进度条 | 青色 | 实时进度 |
| 结尾确认 | 绿色 | 完成信号 |

### 7.4 关键假设（待验证）

| 假设 | 验证方式 | 风险等级 |
|---|---|---|
| 用户愿意花 5 分钟完成问卷 | 种子用户测试完成率 | 中 |
| 简易版 11 题足以捕捉关键认知差异 | 对比简易版和深度版的人设相似度 | 中 |
| 情景探针比直接问偏好更准确 | 用户主观评分"这份人设准确吗" | 高 |
| 人设写入后用户会感受到协作质量提升 | 使用前后对比用户满意度 | 高 |
| 三层转化规则生成的人设语言不会让 AI 产生奇怪行为 | 种子用户实际对话测试 | 中 |

## 8. 发布计划

### 8.1 MVP（第一版，约 2 周）

**目标**：核心功能跑通，支持种子用户测试

- [ ] 简易版问卷（11 题）完整实现
- [ ] 三层转化规则完整实现
- [ ] 人设文件写入（`persona.md`）
- [ ] 按维度局部更新流程
- [ ] 视觉设计（ANSI 颜色、进度条、结尾动画）
- [ ] 首批 10 名种子用户测试

**不包含**：深度版问卷、多人设切换、Web UI

### 8.2 第二版（约 1 个月后）

**目标**：覆盖深度用户需求，提升精度

- [ ] 深度版问卷（31 题）完整实现
- [ ] 完成率优化（基于 MVP 测试数据）
- [ ] 人设可视化摘要（更清晰地展示每个维度的设置）
- [ ] 英文版本

### 8.3 未来探索方向

- **团队级人设**：同一个 AI，根据对话对象（老板 / 同事 / 外部用户）切换风格
- **人设漂移检测**：检测用户行为是否与当前人设不匹配，主动提示更新
- **角色市场**：分享和使用社区创建的人设模板

### 8.4 明确不做的事

- 不提供 Web UI 或独立应用（聚焦 Claude Code 生态）
- 不支持多套人设并行切换（一个用户一套，保持简洁）
- 不收集或上传用户数据，所有内容本地存储
- 不依赖 Python、Node.js 或任何运行时环境
