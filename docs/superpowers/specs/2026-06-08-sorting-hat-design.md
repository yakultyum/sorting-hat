## 一、背景与定位

Anthropic 2026 年发表的研究《Emotion Concepts and their Function in a Large Language Model》表明，Claude 内部存在 171 种可测量的情绪表征，这些表征会真实影响 AI 的输出行为。这意味着 AI 的"人格设定"不只是表面风格，而是从底层影响 AI 的沟通方式和判断倾向。

心理学研究也告诉我们，每个人的认知风格、决策模式、信息处理方式都不同——这些差异决定了什么样的 AI 人设最适合你，更像日常生活中你会遇到的性格相似或互补的亲人和朋友，更能和你在工作中碰撞出灵感的火花。

**Sorting Hat** 是一个 Claude Code skill，灵感来自哈利波特的分院帽。它通过结构化问卷帮助用户找到最适配自己工作风格的 AI 人设，并将结果写入本地记忆文件，让 AI 在每次对话中自动以该人设工作。

**目标用户：** 在日常工作中需要频繁与 AI 协作的员工（支付业务部及其他团队）。

## 二、理论框架

问卷融合七个心理学和认知科学框架，每层解决不同问题：

| 理论框架 | 解决的问题 | 覆盖题目 |
|---|---|---|
| **Anthropic 情绪研究** | AI 内部心理状态的情绪底色 | 简易 Q4/Q5，复杂 Q14/Q15 |
| **荣格八维认知功能** | 思维、情感、直觉、感觉的偏好方向 | 简易 Q1/Q2/Q3，复杂 Q6-Q13 |
| **OCEAN 大五人格** | 补 MBTI 盲区（尤其是神经质维度） | 复杂 Q16-Q19 |
| **依恋理论** | 与 AI 的关系模式和连续性需求 | 复杂 Q20-Q23 |
| **认知负荷理论** | 信息密度、格式、脚手架偏好 | 简易 Q6a/Q6b，复杂 Q24-Q26 |
| **调节焦点理论** | 机会导向 vs 风险导向 | 简易 Q7，复杂 Q27 |
| **认知需求 + 控制点** | 参与推理的深度意愿 + 决策控制权 | 简易 Q1，复杂 Q28/Q29 |

## 三、核心功能

### 3.1 触发逻辑

用户调用 `/sorting-hat` 后：

1. **首次使用**（`~/.claude/memory/persona.md` 不存在）：进入完整问卷流程
2. **再次使用**（文件已存在）：展示当前人设摘要 + 各维度设置，通过 `AskUserQuestion` 让用户选择要修改哪个维度，只重新问该维度的问题，更新文件对应部分

### 3.2 两个版本

| | 简易版 | 复杂版 |
|---|---|---|
| 调用方式 | `/sorting-hat` 或 `/sorting-hat lite` | `/sorting-hat full` |
| 题目数量 | 11 题 | 31 题 |
| 耗时 | 约 5 分钟 | 约 15-20 分钟 |
| 适合场景 | 快速上手，立即投入使用 | 深度定制，精确人设 |
| 理论覆盖 | 荣格八维核心 + Anthropic 论文 + 认知负荷 + 调节焦点 | 全部七个框架 |

### 3.3 输出文件

写入 `~/.claude/memory/persona.md`，分两部分：

1. **答题记录**——结构化存储每道题的选择，供下次修改时展示
2. **AI 人设描述**——供 Claude 直接读取的人设 prompt，Claude Code 的 memory 系统自动注入每次对话

## 四、交互规范

### 4.1 选题交互方式

每道选择题通过 `AskUserQuestion` 工具呈现，用户直接点击选项，无需手动输入字母：

- **单选题**：`multiSelect: false`，列出所有选项
- **开放题**：直接在对话中等待用户文字输入
- **可跳过的题**：在选项中加入「跳过」作为最后一个选项

### 4.2 终端视觉设计

使用 ANSI 颜色码渲染，零额外依赖，Claude Code 终端天然支持：

| 元素 | 颜色 | 用途 |
|---|---|---|
| ASCII 大标题 | 金色（`\033[38;5;220m`） | 开场标题 |
| 🎩 emoji | — | 标题下方，仪式感 |
| Phase 标题 | 紫色（`\033[38;5;135m`） | 区分阶段 |
| 题目编号 | 金色 | 视觉层次 |
| 选项字母 | 青色（`\033[38;5;87m`） | 引导用户 |
| 进度条 | 青色 | 实时进度 |
| 分隔符 | 灰色（`\033[38;5;245m`） | 结构感 |
| 结尾确认 | 绿色（`\033[38;5;82m`） | 完成信号 |

**进度条格式：**
```
分院进度 ████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ 20%  Phase 1/5
```

**结尾动画感（逐行输出）：**
```
🔮  正在分析你的答案...
✨  匹配人格图谱...
🎩  生成专属人设...
✓   已写入 ~/.claude/memory/persona.md
```

### 4.3 心理转化规则

问卷的题目是情景探针，目的是挖掘用户选择背后的心理特征，生成人设时必须按三层转化：

```
用户答案 → 心理特征（该答案揭示的认知/情感/行为模式）→ 人设语言（翻译成 AI 的通用行事原则）
```

**错误示例：** 用户选了「带我一步步推导」→ 人设写「你会带用户一步步推导」（直接复制情景）

**正确示例：** 用户选了「带我一步步推导」→ 心理特征：Ti 主导 + 高认知需求，重视理解过程本身 → 人设语言：「暴露思维链，不替用户跳过推理步骤；用户真正理解比快速给出答案更重要」

每道题的完整转化规则详见问卷文档：`docs/questionnaire/questionnaire.md`

## 五、详细流程

### 5.1 首次分院流程

```
调用 /sorting-hat
        ↓
展示 ASCII 标题 + 🎩 + 欢迎语
        ↓
询问版本：简易版 or 深度版（AskUserQuestion）
        ↓
逐题问答（AskUserQuestion 呈现选项）
进度条实时更新
        ↓
汇总所有答案
按三层转化规则生成人设描述
        ↓
写入 ~/.claude/memory/persona.md
（答题记录 + AI 人设描述两部分）
        ↓
展示结尾动画 + 生成结果摘要
```

### 5.2 修改流程

```
调用 /sorting-hat
        ↓
读取 persona.md，展示当前设置摘要
        ↓
AskUserQuestion：要修改哪个维度？
（列出所有维度，也可选"全部重来"）
        ↓
只重新问用户选择的维度
        ↓
按三层转化规则重新生成该维度的人设语言
        ↓
更新 persona.md 对应部分
        ↓
展示更新确认 + 新人设摘要
```

## 七、实现方式

- **类型：** 纯 Markdown skill（`SKILL.md`）
- **依赖：** 零外部依赖，仅需 Claude Code
- **交互：** 通过 `AskUserQuestion` 工具呈现选项，用户点击选择，无需手动输入
- **视觉：** ANSI 颜色码，Claude 直接在回复中输出，终端自动渲染
- **文件写入：** 通过 Claude Code 的 Write 工具写入 `~/.claude/memory/persona.md`
- **文件读取：** Claude Code 的 memory 系统自动加载 `~/.claude/memory/` 下所有文件，无需额外配置
- **存储位置：** `~/.claude/skills/sorting-hat/SKILL.md`
- **问卷文档：** `docs/questionnaire/questionnaire.md`（含完整题目和心理转化规则）

---

## 八、不在范围内

- 不提供 Web UI 或独立应用
- 不支持多套人设切换（一个用户一套）
- 不收集或上传用户数据，所有内容本地存储
- 不依赖 Python、Node.js 或任何运行时环境
